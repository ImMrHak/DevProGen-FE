Welcome to DevProGenerator

DevProGenerator is a robust tool designed to streamline the creation of fully functional projects using Spring Boot for the backend and Angular for the frontend. By defining your entities, fields, relationships, and access controls in a simple YAML file, you can quickly generate the skeleton of your application and focus on building features. You can create as many entities as you need to build your application.

Documentation

Entity Definition YAML

  
  entities:    
    - name: "EntityName"
      fields:
        - name: "fieldName"
          type: "FieldType"
          required: true/false
        # Add more fields as needed
      relationships:
        - name: "relationshipName"
          type: "RelationshipType"
          targetEntity: "TargetEntityName"
          required: true/false
        # Add more relationships as needed
      accessControl:
        - role: "ADMIN"
          allowed: true/false
      inheritanceControl:
        - inheritance: true/false
  # Add more entities as needed

Explanation

  • name: The unique identifier for the entity. This will be used as the table name in the database and the primary reference throughout your application.
  • fields: A list of attributes associated with the entity. Each field entry should include:
    • name: The name of the field (e.g., username, email).
    • type: The data type of the field (e.g., String, Integer).
    • required: A boolean value indicating whether this field is required. If set to true, the field will be marked as required in the generated HTML form, meaning users must provide a value for this field before submitting the form. If set to false, the field will be optional.
  • relationships: Defines how this entity relates to other entities in your application. Each relationship entry should include:
    • name: The name of the relationship (e.g., userRoles, orders).
    • type: The type of relationship (e.g., OneToOne, OneToMany, ManyToOne, ManyToMany).
    • targetEntity: The entity that this relationship points to (e.g., Role, Order).
    • required: A boolean value indicating whether this relationship is required. If set to true, the relationship must be defined and cannot be left empty.
  • accessControl: Specifies the permissions for different roles to access this entity. Each access control entry should include:
    • role: The role for which the access control applies (e.g., ADMIN).
    • allowed: A boolean value indicating whether the role is allowed to access the entity (e.g., true or false).
  • inheritanceControl: Indicates whether this entity inherits properties from an existing User entity. When inheritance is set to true, the entity will inherit all attributes from the User entity, including:
    • id: The unique identifier of the user.
    • firstName: The first name of the user.
    • lastName: The last name of the user.
    • email: The email address of the user.
    • userName: The username of the user.
    • password: The password of the user.
    • roles: A collection of roles associated with the user.
